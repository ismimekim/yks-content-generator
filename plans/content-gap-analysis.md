# İçerik Üretim Boşluğu Analizi

## 📊 ÜRETILEN vs GERÇEK OĞUZ USTA

### Aldığın Çıktı (Sistem)
```
Gençler bu videoyu izleyin çünkü AYT tarih için son 3 ay stratejisi vereceğim.

Bak şimdi, çoğunuz şu hatayı yapıyorsunuz...
[5 madde]
Görüşürüz 💜
```

### Gerçek Oğuz Usta (Referans)
```
Gençler selamlar herkese. Bugün farklı bir video ile karşınızdayım.

AYT 2025 NASILDI DERS DERS YORUMLARIMIZ
[Sınavı değerlendirme]
Sizler yapıyor musunuz bunu? [Empati sorusu]
[Gerçek hikayeleri, öğrenci örnekleri]
Hani şöyle... [Açıklama]
Yani... [Pekiştirme]
```

## 🔍 DERİN FARK ANALİZİ

### 1. TONAL FARKLAR (Ses/Tutum)

| Boyut | Sistem Çıktısı | Gerçek Oğuz | Problem |
|-------|-----------------|------------|---------|
| **Samiimiyet** | "Tarih sadece ezber değil" | "Valla şunu söyleyeyim" | Formal yapı, konuşma tarzı yok |
| **Empati** | "Çoğunuz şu hatayı yapıyorsunuz" | "Biliyorsunuz çok kişi yanlış yapıyor, ben de yaptım" | Kişisel hikaye eksik |
| **Aciliyet** | "3 ay var, sistemli çalış" | "3 ay var daha, panik yapma. Sistemli çalış." | Kaygı + Motivasyon dengesi zayıf |
| **Kişilendirme** | Genel tavsiye | "Sen" hitabı, kişisel deneyim | Yok hükmünde kişiselleştirme |

### 2. YAPISAL FARKLAR (Akış/Organizasyon)

| Boyut | Sistem | Gerçek | Gap |
|-------|--------|--------|-----|
| **Giriş** | Doğrudan konu | Problem tanımlama + Empati | Başında soru/şok eksik |
| **Madde Yapısı** | CAP LOCK başlık, açıklama | CAP LOCK başlık, "Hani şöyle..." açıklama, örnek, vurgu | Açıklamada derinlik yok |
| **Geçişler** | Makine gibi | "Şimdi gençler...", "Peki..." | Doğal akış eksik |
| **Pekiştirme** | 1 kez söyleniyor | 3-4 kez farklı açılardan | Tekrar mekanizması yok |
| **Kapanış** | Görüşürüz 💜 | Görüşürüz 💜 + Motivasyon | Motivasyon cümlesi eksik |

### 3. İÇERİK DERINLEME FARKLARI (YKS Know-How)

| Boyut | Sistem | Gerçek | Gap |
|-------|--------|--------|-----|
| **Spesifisitesi** | Genel taktik | ÖSYM spesifik (sebep-sonuç, kronoloji) | Sınav hazırlığı taktikleri yok |
| **Alternatif Kaynaklar** | Genel | Palme, Hız ve Renk, Limit | Brand bilgisi eksik |
| **Zaman Yönetimi** | Genel | "Mayıs'ta konuyu öğrenme zamanı değil" | Kritik zaman noktaları yok |
| **Problem Çözme** | Taktik | "Hani şöyle yapıyorlar..." + çözüm | Sorunu görmek yok |
| **ÖSYM Stratejisi** | Yok | "Sebep-sonuç ilişkileri, kronoloji, karşılaştırma" | Doğrudan yok |

### 4. EMOSYONEL FARKLAR (Hissiyat)

| Eleman | Sistem | Gerçek |
|--------|--------|--------|
| **Motivasyon Kaynağı** | Mantık | Umut + Aciliyet + Empati |
| **Öğrenci Konumu** | "Siz" (uzakta) | "Sen" (yakın) |
| **Başarı Vurgusu** | "Sistemli çalış" | "Başaracaksın, biliyor musun neden?" |
| **Başarısızlık Tedavisi** | "Yapma" | "Yaptığını anlıyorum ama işte bu yüzden..." |

### 5. TEKNİK KALIP FARKLARI

**Eksik Oğuz Kalıpları:**
- [ ] "Hani şöyle..." (açıklama)
- [ ] "Yani..." (pekiştirme)
- [ ] "Valla şunu söyleyeyim" (samimi açıklama)
- [ ] "Asla... Kesinlikle..." (vurgu)
- [ ] "Şimdi gençler..." (geçiş)
- [ ] "Peki..." (geçiş)
- [ ] "Biliyorsunuz..." (empati)
- [ ] Soru-Cevap Format (İç Diyalog)

  --output data/style_profiles/oguz_usta_patterns.json \

**Çıktı**: JSON profile
```json

  "opening_formulas": [...],
  "problem_patterns": [...],
  "explanation_markers": ["hani şöyle", "yani", "valla"],
  "emphasis_words": ["asla", "kesinlikle", "mutlaka"],
  "transitions": [...],
  "body_structure": "madde + hani şöyle + örnek + vurgu",
  "closing_formulas": [...]
}
```

### Phase 2: Content Mimarı
```typescript
// lib/content-builder.ts
interface ContentRequest {
  topic: string;
  platform: "youtube" | "tiktok";
  targetAudience: "11ler" | "12ler" | "mezun";
  urgency: "low" | "medium" | "high";
  styleProfile: StyleProfile;
}

function buildStructuredContent(req: ContentRequest): StructuredContent {
  return {
    sections: [
      {
        type: "opening",
        formula: selectOpening(req.targetAudience),
        content: generateOpening(req.topic)
      },
      {
        type: "problem_intro",
        pattern: "çoğunuz şu hatayı yapıyorsunuz",
        problem: identifyProblem(req.topic),
        empathy: addEmpathy(req.targetAudience)
      },
      {
        type: "main_content",
        maddeCount: req.urgency === "high" ? 7 : 5,
        structure: "başlık + açıklama + örnek + vurgu"
      },
      {
        type: "closing",
        formula: "Görüşürüz 💜",
        motivation: addMotivation(req.urgency)
      }
    ]
  };
}
```

### Phase 3: LLM Polish
```python
# scripts/refine_content.py
def refine_with_llm(structured_content, style_profile):
    prompt = f"""
    Bu yapılı içeriği {style_profile['source']} tarzında yaz.
    
    KURALı:
    1. Kalıpları koru: {style_profile['opening_formulas']}
    2. Açıklamalarda "Hani şöyle...", "Yani..." kullan
    3. Vurguya "Asla", "Kesinlikle" ekle
    4. Geçişte "Peki...", "Şimdi gençler..." kullan
    5. Motivasyon ekle: Urgency={content['urgency']}
    6. Empati göster: Hedef = {content['targetAudience']}
    
    YAPMAZ:
    - Kalıplar dışında kelime kullanma
    - Formal yapı koru ama konuşma tarzı yap
    - Sahte örnekler ekleme
    
    STRUKTÜRELİ İÇERİK:
    {structured_content}
    """
    
    return claude_api.call(prompt, max_tokens=2000)
```

## 📋 ÖZET: Sistem Nasıl Olmalı?

1. **Kalıpları Doğru Çıkar** → Gerçek Oğuz Usta metinlerinden
2. **Yapı Mimarı** → Sections + Formula matching
3. **LLM Alet Olsun** → Kalıpları koruyarak dokunuş yap
4. **Validasyon** → "Bu Oğuz mu?" test
5. **YKS Know-How** → Sistem tarafından, LLM'ye değil

**Kritik**: LLM'ye "Oğuz Usta gibi yaz" değil, "Bu kalıpları koru, bu akışı takip et" demelisin.
